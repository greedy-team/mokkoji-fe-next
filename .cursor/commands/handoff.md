# 컨텍스트 핸드오프

새로운 AI 세션이 즉시 작업을 이어받을 수 있도록 핸드오프 문서를 생성한다.

## 실행 절차

1. `git diff --stat`과 `git status`로 현재 변경 상태를 확인한다.
2. 아래 형식에 맞춰 핸드오프 내용을 **채팅에 직접 출력**한다. 파일을 생성하지 않는다.
3. 사용자가 복사하여 새 세션 첫 메시지에 붙여넣을 수 있도록 코드블록으로 감싼다.

## 핸드오프 문서 형식

```markdown
# Handoff: {작업 제목}

> 생성: {타임스탬프} | 브랜치: {브랜치명}

## 목표
{이 세션에서 달성하려는 최종 목표. 한 문장.}

## 완료된 작업
{번호 매긴 목록. 각 항목에 변경한 파일 경로 포함.}

## 미완료 작업
{번호 매긴 목록. 각 항목에 예상 접근 방식 포함.}

## 핵심 결정 사항
{이 세션에서 내린 설계/구현 결정과 그 이유. 새 AI가 같은 결정을 반복하지 않도록.}

## 현재 상태
- 빌드: {성공/실패 + 에러 요약}
- 변경 파일: {git status 요약}
- 커밋 여부: {마지막 커밋 해시 또는 "미커밋"}

## 주요 파일 맵
{이 작업과 관련된 핵심 파일 경로 목록. 각 파일의 역할을 한 줄로 설명.}

## 주의사항
{새 AI가 반드시 알아야 할 제약, 버그, 함정.}
```

## 규칙

- 핸드오프 문서는 **AI가 읽는 문서**다. AI가 가장 효율적으로 파악할 수 있게 작성한다.
- 모호한 표현 대신 구체적인 파일 경로, 함수명, 에러 메시지를 포함한다.
- "~한 것 같다", "아마도" 같은 불확실한 표현을 사용하지 않는다.
- 불필요한 배경 설명 없이 사실만 나열한다.
